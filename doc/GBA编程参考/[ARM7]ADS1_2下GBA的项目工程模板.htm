[ARM7]ADS1.2下GBA的项目工程模板
作者：mail2li

论坛FTP的/文件上载区/ADS1.2 GBA 工程模板_by mail2li.zip
或者直接下载这个文件：upload/03111717111109.zip
包含一个代码样例 和 辅助工具 方便开发
将文件解压到ADS安装目录的Stationery下 new一个工程的时候就自动增加了一个项目
：-------------------------------------------------------：
        GBA MULTIBOOT 模式开发ADS1.2工程模板
将压缩包解压拷贝到“C:\Program Files\ARM\ADSv1_2\Stationery\”

（假设你的ADS1.2安装于此目录下）
启动ADS1.2的IDE 在新建工程的时候 会出现一个“AGB MULTIBOOT”项

其他就enjoy it
我的main()中已经包含了一个显示内存数据的代码 作为参考还有好玩

哈哈……
其他：
1.包含一个在BG3模式下的英文字符格式化输出函数 printLCD();
格式化输出列表同标准C兼容 "%???"... 具体可以参考我的程序
核心是cons结构体 依照范例修改 可以使用在所有的字符型的输入输出

终端上：LCD,UART,KEY,IIC,SPI...
可以在几乎所有的ANSI C编译器上编译
原本的printk函数是一个gcc下的源码 可以简化修改在KEIL C51下使用（事实

上我就在用 ^_^ ）
2.GBA.H文件 基于HAM的mygba.h简化 其中部分寄存器名依《AGB Programming
Manual Ver1.1》更正 但不完全希望使用者自行测试和更新
3.TOOLS目录下工具 版权归原作者
gbafix.exe
修正*.GBA的头部结构 要你的程序能在GBA上跑 必须保证一定的

GBA头结构 具体参见文档
VisualBoyAdvance.exe
GBA模拟器 你的程序可以先在模拟器下模拟运行
XBOO.COM       
并口下载 下载线做法参见XBOO.txt文件
4.字库支持三种 6X12,8X12,8X16 通过在driver.h中的头文件中定义 ASC_FONT_W
ASC_FONT_H 两宏切换
取字模是用的龙啸九天的ZM3 欢迎大家购买（www.C51BBS.com）――龙
